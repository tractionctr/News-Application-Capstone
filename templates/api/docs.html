API Documentation

Authentication

  • POST /api/token/
  • Token-based authentication required for all endpoints

Articles

  • GET /api/articles/ → All approved articles
  • GET /api/articles/<id>/ → Single article
  • GET /api/articles/subscribed/ → Subscribed content
  • POST /api/articles/ → Create article (Journalist only)
  • PUT /api/articles/<id>/ → Update (Journalist/Editor)
  • DELETE /api/articles/<id>/ → Delete (Journalist/Editor)

Newsletters

  • GET /api/newsletters/
  • GET /api/newsletters/<id>/
  • POST /api/newsletters/ → Journalist/Editor
  • PUT /api/newsletters/<id>/
  • DELETE /api/newsletters/<id>/

Roles & Permissions

  • Reader → View only
  • Journalist → Create + manage own content
  • Editor → Full control + approvals

Example

GET /api/articles/

  {

    "title": "Breaking News",
    "content": "Hello world"
